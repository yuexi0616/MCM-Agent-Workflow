# 动态规划多阶段决策

将复杂生产流程分解为顺序决策阶段，从后向前递推最优决策。2024 B 题是 DP 的经典应用场景。

## MCM 应用（2024 B 题）

### 多工序生产系统
1. 购买零配件→检查→装配（半成品1）→检查→装配（半成品2）→...→装配（成品）→检查→销售
2. 每个阶段决策：是否检测该阶段的输入件/半成品
3. 不合格品决策：丢弃 or 拆解

### 状态-决策模型（B159）
- **状态**：当前零件的合格/不合格、是否已检测、来自哪个上游阶段
- **决策**：检测/不检测、拆解/不拆解（0/1 变量）
- **递推关系**：$V(s) = \max_{a} \{ R(s,a) + \sum_{s'} P(s'|s,a) V(s') \}$
- **记忆化**：缓存中间结果，避免重复计算

### 全概率迭代（B195）
- 将拆解决策转化为不同阶段参数
- 所有决策转化为概率事件→净利润比较
- 16 个决策变量→遍历所有 0/1 组合（$2^{16}=65536$）或模拟退火

## 通用框架

1. 定义阶段（stage）和状态（state）
2. 定义每个阶段的可选决策（action）
3. 定义状态转移和即时收益 $R(s,a)$
4. 从终端状态向后递推→Bellman 最优方程
5. 提取最优策略（policy）
