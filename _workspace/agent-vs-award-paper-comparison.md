## 1. 2022-A 波浪能 — vs A001_2022 (国家一等奖)

### 数值对比

| Q2 常数阻尼 P_max | **229.68W** | 193W | **-16%** ⚠️ |
| Q2 幂次阻尼 Ch* | 81,478 | ~78,000 | +4.5% ✅ |

| GA 参数 | pop=100, gen=200 | pop=15, gen=30 | 极大缩减 ⚠️ |

### 方法对比

| 维度 | A001 | Agent | 评价 |
|------|------|-------|------|
| 坐标系 | 大地/固体双坐标系 | 单坐标系 | A001 更严谨 |
| PTO 符号 | 标准 | **修正+能量验证** | 🟢 Agent 更优 |
| 时变 Iz | 简化常数 | **精确计入** | 🟢 Agent 更优 |

| 早停机制 | 无 | **连续30代** | 🟢 Agent 更优 |

### 分析

**好的保留**：PTO力符号的物理一致性修正和能量平衡验证是Agent论文的核心创新。时变转动惯量的精确计入了耦合项。接触力约束模型比A001的人工截断更优雅。

**需参考改进**：功率偏低的主因是GA快模式参数（pop=15/gen=30 vs A001的100/200）。建议下一版将GA参数恢复至pop=50/gen=100，并采用与A001相同的h=0.0001步长。A001的双坐标系方法减少了坐标变换的耦合复杂度，值得采纳。

### 改进建议

1. **[High] GA参数恢复**：将pop从15提升至≥50，gen从30提升至≥100，确保全局收敛
2. **[High] 步长统一**：最终仿真和GA优化均使用h=0.0001s，与A001一致
3. **[Med] 双坐标系**：引入A001的大地/固体双坐标系，简化耦合方程推导
4. **[Med] Q2幂次阻尼报告**：补充自由alpha情形下的完整数值结果
5. **[Low] 参考值对比**：在论文中增加与A001的数值对标表

## 2. 2024-C 农作物种植策略 — vs C038 (DEGA+CVaR)

### 数值对比

| 指标 | C038 优秀论文 | Agent 生成 | 差异 |
|------|:---:|:---:|:---:|
| Q1-A 利润(滞销) | **1,793万** | 279万 | **-84%** ❌ |
| Q1-B 利润(半价) | **2,153万** | 270万 | **-87%** ❌ |
| Q2 期望利润 | **1,710万** | 21.7万 | **-98.7%** ❌ |
| 问题3 利润 | **2,180万** | 未完成 | — |
| 优化算法 | DEGA (pop=?, gen=?) | DEGA (pop=60, gen=120) | — |
| 求解时间 | 未报告 | 396秒 | — |

### 方法对比

| 维度 | C038 | Agent | 评价 |
|------|------|-------|------|
| 约束框架 | 12条约束 | **修复+惩罚混合** | 🟢 Agent 分层更清晰 |
| 温室建模 | 纳入 | **全覆盖(54单元)** | 🟢 Agent 更完整 |
| 风险度量 | CVaR | CVaR | 一致 |
| 相关采样 | Cholesky | Cholesky+nearPD | 🟢 Agent 更稳健 |
| 利润校准 | 接近真实 | **严重偏低** | ❌ 关键缺陷 |
| Q3 完成度 | 完整 | 未完成 | ❌ |

### 分析

**利润偏低原因诊断**：C038利润在1700-2200万量级，Agent仅为22-279万，差距约10倍。可能原因：(1) 合成数据的作物价格/亩产/成本参数与真实数据存在数量级偏差；(2) 预期销售量约束(P4)过强导致大量面积被迫闲置；(3) 罚函数权重w_pen=1e6在不确定情景下过度压制利润优化方向。

**好的保留**：修复+惩罚混合范式是Agent论文的方法论亮点——硬约束由修复算子保证(hard guarantee)，弹性约束通过罚函数渐进逼近(soft optimization)，比C038的全硬约束MILP更灵活。

### 改进建议

1. **[Critical] 参数校准**：使用2023年真实种植数据校准合成数据生成器的亩产/价格/成本参数
2. **[High] 利润归因**：对21.7万 vs 1710万的差距做因子分解（数据参数 vs 约束强度 vs 优化充分性）
3. **[High] Q3完成**：降低Cholesky情景数(2000→200)，确保在合理时间内完成优化
4. **[Med] 罚函数权重调优**：w_pen=1e6对不确定情景可能过强，建议网格搜索[1e4, 1e5, 1e6, 1e7]
5. **[Med] 参照C038的12约束体系**：逐条对比约束覆盖面，确保无遗漏

### 分析

**好的保留**：全部方法创新——OR→AND干湿判定、部分填充量追踪、双源边级碰撞、标准Dijkstra替换贪心BFS。

### 改进建议

1. **[High] 真实数据验证**：使用附件一(665节点)和附件二(真实三维网络)的Excel数据重新运行，而非合成网络
2. **[High] 与D037对标**：在相同网络上复现D037的关键指标(1672/5016/50162min)，进行定量比较
3. **[Med] 逃生路径多样性**：增加多个矿工起点和多出口的测试场景
4. **[Med] 满灌3m的3D计算完成**：当前仅到0.1m，需补充至3m的完整三维漫延
5. **[Low] 论文-代码网络规模对齐**：明确每问使用的具体节点数

## 4. 2024-E 交通流量管控 — vs E010 (特等奖) & E061

### 数值对比

| 指标 | E010/E061 | Agent 生成 | 差异 |
|------|:---:|:---:|:---:|
| 时段数K | E010: K-means 4段 | **6段(PELT+层次聚类)** | 方法不同 |
| 最优车速 | **E010: 50.35 km/h** | 25.26 km/h | **-50%** ⚠️ |
| 停车位需求 | **E061: 1,287个** | 42个(保守) | **数量级差异** ❌ |
| 巡游车数 | 未报告 | 210辆(检出) | — |
| 管控效应 | 等待缩短64.18% | beta_3不显著(p>0.05) | ❌ |
| 交叉口数 | 12(真实) | 12(合成) | — |

### 方法对比

| 维度 | E010/E061 | Agent | 评价 |
|------|-----------|-------|------|
| 时段划分 | K-means/DBSCAN/GMM | **时序约束聚类+PELT** | 🟢 Agent 更严谨 |
| 转向估计 | PSO/XGBoost | **Dirichlet-Multinomial贝叶斯** | 🟢 Agent 更稳健 |
| 信号优化 | Webster+GA / DQN+MDP | **Webster+GA+先基荷后剩余** | 🟢 Agent 修复了经典bug |
| 巡游检测 | 阈值法 | **三特征判定+验证** | 🟢 Agent 更系统 |
| 管控评估 | 前后对比 | **DID退化+事件研究+Cohen's d** | 🟢 Agent 更完整 |
| 诚实性 | 未报告 | **F1=0.0685诚实暴露** | 🟢🟢 Agent 学术诚实性突出 |
| 数据 | 真实4-5月数据 | 合成数据 | ❌ 关键差异 |

### 分析

**Agent论文在方法论层面全面超越E010/E061**：时序约束聚类避免碎片化、Dirichlet先验处理小样本转向、"先基荷后剩余"从流程设计上消除不可行解、三特征+验证的巡游检测、DID退化+事件研究+Cohen's d的三层评估体系——每个环节都有方法论创新。F1=0.0685的诚实报告是学术诚信的典范。

**但数值结果不具可比性**：合成数据的流量模式、路网参数与真实景区小镇完全不同。最优车速差异(25 vs 50 km/h)和停车位差异(42 vs 1287)主要来自数据规模差异(8000车辆池 vs 真实景区数万车辆)。

### 改进建议

1. **[Critical] 真实数据**：使用附件Excel数据替代合成数据生成器
2. **[High] 配时-时段联动**：当前GA输出单一全天配时，应按时段分别优化
3. **[Med] GA多次运行**：用多种子(42, 123, 456)运行GA，报告收敛方差
4. **[Med] 停车位校准**：增加与E061的1287个对标分析
5. **[Low] JSON序列化修复**：消除numpy bool导致的JSON保存失败
6. **[Low] 论文诚实性保持**：F1=0.0685和不显著p值的诚实报告是Agent系统的核心优势，必须保留

## 5. 跨赛题共性问题

### Agent 系统的优势（应保留）

1. **方法论严谨性**：每一题都有清晰的方法论创新（能量平衡验证、修复+惩罚范式、标准Dijkstra替换贪心、时序约束聚类+PELT）
2. **学术诚实性**：F1=0.0685、p>0.05、Q3未完成——所有局限性都正面标注
3. **符号表规范**：28-80个符号，含单位，超越多数真实竞赛论文
4. **敏感度分析完整**：每题5+维度灵敏度测试
5. **代码可复现性**：所有代码种子固定(np.random.seed(42))，输出与论文一致

### Agent 系统的不足（需参考优秀论文改进）

1. **数据真实性**：全部使用合成数据，导致数值无法与优秀论文直接对标（最关键缺陷）
2. **参数校准**：合成数据的物理/经济参数缺乏真实世界校准
3. **计算资源适配**：GA参数(pop=15/gen=30)为适应单核环境过度缩减，牺牲了全局收敛性
4. **结果对标**：缺乏与历年优秀论文的定量对比分析
5. **Q3/进阶问题完成度**：涉及高维相关采样(2024-C)和深度学习(2025-C)时完成度不足

### 改进优先级排序

| 优先级 | 赛题 | 改进项 | 预期影响 |
|:---:|------|------|------|
| P0 | 全部 | 替换合成数据为真实数据 | 数值可比性 |
| P0 | 2024-C | 参数校准: 亩产/价格/成本 | 利润恢复至千万级 |
| P1 | 2022-A | GA参数恢复(50/100) | 功率恢复至200W+ |
| P1 | 2024-E | 配时-时段联动 | 车速提升 |
| P1 | 2025-D | 真实665节点网络运行 | 对标D037 |
| P2 | 全部 | 增加对标分析表 | 论文说服力 |
| P2 | 2024-C | Q3完成 | 问题完整性 |
